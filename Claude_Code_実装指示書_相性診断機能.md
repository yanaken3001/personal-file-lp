# パーソナルファイル 相性診断機能 実装指示書

## 概要

性格診断サービス「パーソナルファイル」の相性診断機能を実装する。
16タイプの性格診断結果をもとに、2人のユーザー間の相性スコアと解説を表示し、
SNSでシェアできる相性カードを生成する機能。

## 2. 相性リンク型バイラルフロー

### 2-1. ユーザーフロー全体図

```
[Aさん] 診断完了
  → 結果画面に [友達との相性を調べる] ボタン表示
  → タップで「相性リンク」を生成
     URL例: personalfile.jp/match/{unique_id}
     unique_id にAさんの16タイプ（例: "PP"）と行動類型（例: "achievement"）を紐づけて保存
  → AさんがLINE / DM / SNSでリンクを共有

[Bさん] リンクを開く
  → ランディング画面:「Aさん（チームの太陽）があなたとの相性を知りたがっています」
  → Bさんが診断を受ける
  → Bさんの結果 + Aさんとの相性結果が同時に表示される
  → 相性カード（画像）が自動生成される
  → [この相性をシェアする] [他の友達との相性も調べる] ボタン表示
  → Aさんにも通知（LINE or メール）:「Bさんとの相性が出ました！」
```

### 2-2. データベース設計

```
テーブル: match_links
  - id: UUID (primary key)
  - creator_user_id: ユーザーID（Aさん）
  - creator_type_code: VARCHAR(2) -- "PP", "AD" 等
  - creator_behavior_code: VARCHAR(20) -- "achievement" 等
  - created_at: TIMESTAMP
  - expires_at: TIMESTAMP -- 30日で期限切れ

テーブル: match_results
  - id: UUID (primary key)
  - match_link_id: FK → match_links.id
  - responder_user_id: ユーザーID（Bさん）
  - responder_type_code: VARCHAR(2)
  - responder_behavior_code: VARCHAR(20)
  - compatibility_score_16: INTEGER -- 16タイプレベルのスコア（54-98）
  - compatibility_label: VARCHAR(20) -- "運命の相棒" 等
  - relationship_type: VARCHAR(10) -- "共感" / "補完" / "共感＋補完" / "理論値"
  - created_at: TIMESTAMP
```

### 2-3. 相性スコア算出ロジック

```
function getCompatibility(typeA, typeB) {
  // Step 1: 16タイプレベルのスコアをマトリクスから取得
  const score16 = COMPATIBILITY_MATRIX[typeA][typeB];
  
  // Step 2: ラベルを判定
  const label = LABELS.find(l => score16 >= l.min && score16 <= l.max);
  
  // Step 3: 関係タイプを判定（公式データに共感/補完の情報がある場合）
  const relationType = getRelationType(typeA, typeB);
  
  return {
    score: score16,
    label: label.label,
    icon: label.icon,
    tone: label.tone,
    relationType: relationType
  };
}
```

## 4. 実装上の注意事項

### 4-1. 相性スコアの非対称性

マトリクスは非対称である。つまり `MATRIX["PP"]["AA"] = 98` だが `MATRIX["AA"]["PP"] = 54`。
これは「PPから見たAAとの相性」と「AAから見たPPとの相性」が異なることを意味する。

相性結果画面では**2つのスコアの平均値**を表示する。

```
displayScore = Math.round((MATRIX[typeA][typeB] + MATRIX[typeB][typeA]) / 2)
```

例: PP×AA → (98 + 54) / 2 = 76%

ただし、個別の視点も「さらに詳しく」セクションで表示可能にする：
- 「チームの太陽から見た相性：98%」
- 「歩くWikipediaから見た相性：54%」

### 4-2. OGP（Open Graph Protocol）対応

相性リンクのURLにアクセスしたとき、OGPメタタグで以下を返す：
- og:title → 「チームの太陽があなたとの相性を知りたがっています」
- og:image → Aさんのキャラクターイラスト画像
- og:description → 「無料の性格診断で相性を調べよう」

相性結果のシェアURLにアクセスしたとき：
- og:title → 「チームの太陽 × 氷の頭脳 ⚡運命の相棒 76%」
- og:image → 相性カード画像（自動生成）
- og:description → 「あなたも診断して相性を調べよう」

### 4-3. 相性リンクの制約

- 1つの相性リンクに対して複数人が診断可能（Aさんは1つのリンクで何人とも相性を見れる）
- リンクの有効期限：30日
- 1ユーザーあたりの相性リンク生成上限：なし

### 4-4. パフォーマンス

- 相性スコアの算出はクライアントサイドで完結可能（マトリクスデータはフロントに持たせてOK）
- 相性カード画像の生成はサーバーサイドで行い、CDNにキャッシュする
- キャッシュキー：`{typeA}_{typeB}` の組み合わせで最大 16×16 = 256パターン
