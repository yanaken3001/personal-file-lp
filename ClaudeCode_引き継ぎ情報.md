## 2. 診断システムの全体仕様

### 2-1. 性格類型診断（32問）

**判定ロジック：**
- P / A / I / D の4軸を各8問・5段階評価（1〜5点）で測定
- 最高得点軸 → 基本系統
- 2位軸 → 補助系統
- **1位と2位の差が16点以上 → 純粋類型**（旧仕様の20点基準を8問版に換算）
- 同点時の優先順位：D → P → A → I

**4軸の意味：**

| 軸 | 特性 |
|----|------|
| P軸 | 行動・感覚・楽観志向 |
| A軸 | 分析・慎重・知識志向 |
| I軸 | 協調・受容・チーム志向 |
| D軸 | 決断・独立・挑戦志向 |

### 2-2. 行動類型診断（25問）

**判定ロジック：**
- 達成・平和・情報・外見・効率 の5類型を各5問・5段階評価で測定
- 最高得点類型 → 行動タイプとして採用

### 2-3. 80タイプの構造

```
性格類型16タイプ × 行動類型5タイプ = 80タイプ
```

## 4. 設問設計の原則（変更時に守ること）

設問の最適化で除外・修正した基準。今後設問を追加・変更する際も同じ基準で判断すること。

**除外した設問の理由3パターン：**

1. **冗長設問**：同じことを別の言い方で聞いている設問は1問に集約
2. **交差汚染**：ある軸を測ろうとしているのに別の軸とも強く相関する設問は除外
3. **社会的望ましさバイアス**：「かなりの負けず嫌い」「絶対に損したくない」のような極端な表現は中立的な表現に書き換え

## 6. 極秘マニュアルとアドバイザー管理画面設計

### 6-1. 極秘マニュアル（_極秘__マニュアル__v2.pdf）とは

54ページ、全16タイプ×5行動類型の**「本音の注意点集」**。

ユーザーには絶対に見せないレベルの内容が書いてある。例えば：

```
【外見型PP型について】
「自分をイケている人・選ばれし者と信じ込んでいる
 ナルシスト気質が強く、実力不足のまま年齢を重ねる」

【効率型PA型について】
「どうやったら楽をして稼ぐか、という論点に行き着き
 外回り中にネットカフェで過ごす人がいる」
```

これがそのまま面談の**「あたり感」**の武器になる。

## 7. 価値観・モチベ・メンタル診断の扱い

**Web診断には含めない（確定）。**

理由：
- 価値観・モチベはキャリアアドバイザーが解釈して初めて価値になる
- Web診断で全部出すと面談の価値が下がる
- メンタル診断は医療的文脈で誤解されるリスクがある

**代替設計：**
- 面談登録後に「事前アンケート」として送付
  - 価値観診断：12問（元24問を半減）
  - モチベーション診断：10問（元30問を1/3に削減）
- メンタル診断はアドバイザーが面談中に口頭でヒアリング

## 8. 設問の出題順序

**推奨：性格類型と行動類型を混合したランダム順**

理由：
- 同じ軸の設問が連続すると「回答パターン学習」が起きて精度低下
- 「どのタイプを測っているか」が見えにくくなり社会的望ましさバイアスを抑制

**推奨ブロック構成（例）：**
```
Q1〜Q8   : P軸2問・A軸2問・I軸2問・D軸2問（混合）
Q9〜Q16  : 行動類型（各類型から1〜2問ランダム）
Q17〜Q32 : 性格類型続き（混合）
Q33〜Q57 : 行動類型続き（混合）
```

## 9. 精度に関する重要な数値

| 設計 | 問数/軸 | 軸スコアCronbach α | 完了率推定 |
|------|--------|------------------|-----------|
| 現行 | 20問 | 0.88 | 20〜35% |
| 今回 | 8問 | 0.82（推定） | 65〜70% |

**なぜ8問で妥協したか：**
- 精度を上げるために問数を増やすと完了率が下がり面談数が減る
- 1,000人が診断開始した場合、89問設計より57問設計の方が面談数が約10件多くなる
- 傾向スコアの細分化は不要（Excelに80タイプ×価値観・モチベ傾向別のコンテンツが全部入っているため）

**純粋類型の閾値（未確定・要実装）：**
- 旧仕様：20問で1位と2位の差が20点以上
- 新仕様：8問版での換算値として「差が16点以上」を採用（要検証）
