# パーソナルファイル KPI戦略 & ファネル改善 議論まとめ

## このドキュメントについて

このドキュメントは、以下の一連の議論をまとめたものです。他のAIとの共有を目的としています。

1. X（旧Twitter）の@nom0607によるバズポスト「売上が生まれる一歩手前に設定している、様々な業界のKPI 11選」のファクトチェック
2. そのKPI設計の考え方を、適職診断をフックにした未経験転職サービス「パーソナルファイル」にどう適用するかの戦略議論

## Part 1：業界KPI 11選のファクトチェック結果

### 元ポストの趣旨
「売上そのものではなく、売上が生まれる"一歩手前"のエンゲージメント指標をKPIに設定することで成長した企業11社」の事例紹介。37万インプレッションを獲得したバズポスト。

### 各社のファクトチェック

#### ❶ Dropbox
- **ポストの主張：** 追わなかったKPIは「広告経由の新規登録数」、置いたKPIは「1ユーザーあたりの招待送信数」。招待を送った人は定着することに気づき、ストレージ付与を招待行動に全振り。
- **判定：概ね正しい ✅**
- **事実：** Dropboxは紹介プログラムを成長の中核に据えた。招待した人もされた人も500MBの無料ストレージを獲得。2010年4月には月280万件の招待が送られ、15ヶ月で3900%成長（10万→400万ユーザー）。AdWordsのCPAが$233-388と高すぎたため広告が割に合わなかったという経緯がある。
- **一次ソース：** Drew Houston（CEO）のプレゼン「Startup Lessons Learned」（2010年4月）

#### ❷ Slack
- **ポストの主張：** 追わなかったKPIは「契約社数」、置いたKPIは「1チーム内の週次メッセージ数」。10,000通/チーム/週を超えると解約しないというラインを発見。
- **判定：数字が不正確 ❌**
- **事実：** Stewart Butterfield（CEO）が述べた実際の数字は「チームの**累計2,000メッセージ**」。正確な引用は「2,000メッセージを交換したチームの93%が今もSlackを使い続けている」。50人チームで約10時間分、10人チームで約1週間分のメッセージ量。ポストの「10,000通/チーム/週」は数字（5倍の差）も単位（累計 vs 週次）も間違い。
- **一次ソース：** Stewart Butterfield発言（First Round Review等で引用多数）

#### ❸ Spotify
- **ポストの主張：** 追わなかったKPIは「有料会員数」、置いたKPIは「週1回以上の再生日数」。何日使ったかを徹底管理し、連続利用が切れる前にプレイリスト・通知・レコメンドを差し込む。「課金は後、習慣化が先」。
- **判定：一次ソース未確認 ⚠️**
- **事実：** SpotifyのNorth Star MetricはDAU（Daily Active Users）とされ、公式ブログでは「Time Spent Listening（TSL）」を重要指標として言及。「週1回以上の再生日数」を公式KPIとして発表した一次ソースは未確認。リテンション施策の方向性は正しいが、具体的なKPI指標の記述は推測と判断。

#### ❹ Duolingo
- **ポストの主張：** 追わなかったKPIは「学習完了率」、置いたKPIは「連続学習日数（Streak）」。完璧に覚えさせるというよりも、今日も1分触ったかだけを見る。「成果よりもやめなかった日数を重要視」。
- **判定：概ね正しい ✅**
- **事実：** DuolingoのNorth Star MetricはDAU。Streakが最も強力なエンゲージメント機構であることは元CPO Jorge Mazalが公開記事で明言。「10日間のStreakを達成したユーザーはドロップオフ率が大幅に低下する」。Streak関連で600以上のA/Bテストが実施された。
- **一次ソース：** Jorge Mazal "How Duolingo reignited user growth"（Lenny's Newsletter, 2023年）、Duolingo公式ブログ

#### ❺ Costco
- **ポストの主張：** 追わなかったKPIは「商品粗利」、置いたKPIは「会員更新率」。商品は粗利約10〜15%、売上総利益の多くを年会費収入（営業利益の約7割）。
- **判定：正確 ✅**
- **事実：** Costcoはサードパーティ商品14%、Kirklandブランド15%を粗利上限としてキャップ。会員費は営業利益の65〜98%を占める。会員更新率は米国・カナダで約92〜93%、全世界で約90%。グロスマージンは約10〜12%。
- **一次ソース：** Costco年次報告書（10-K）、投資家向け決算報告

#### ❻ IKEA
- **ポストの主張：** 追わなかったKPIは「購入点数」、置いたKPIは「来店滞在時間」。長く歩かせる導線、途中で飯、最後に小物でファンにさせる仕組みづくり。
- **判定：方向性は正しいが公式KPIかは未確認 ⚠️**
- **事実：** IKEAの迷路型レイアウト戦略、レストラン配置、出口前の小物エリアは広く知られた事実。ただし「KPIとして来店滞在時間を設定」の一次ソース（IKEA公式の発表や経営陣の発言）は未確認。

#### ❼ Amazon
- **ポストの主張：** KPIを短期利益→顧客体験（NPS・配送スピード）に変更。初期は利益度外視で配送の速さ、返品のしやすさ、レビューの充実を徹底。
- **判定：概ね正しい ✅**
- **事実：** ジェフ・ベゾスが利益よりも顧客体験を優先したのは有名。株主への手紙でも繰り返し語られている。ただし「NPSをKPIに置いた」の具体的一次ソースは未確認。

#### ❽ Netflix
- **ポストの主張：** KPIを新規契約数→視聴継続時間に変更。広告で契約を取るより「今日は何分観たか」を最重要指標に。
- **判定：概ね正しい ✅**
- **事実：** Netflixは2022年から四半期決算で「視聴時間（Viewing Hours）」を主要指標として公表。「視聴時間が伸びると解約率が下がる」という関係性もNetflix経営陣が複数回言及。

#### ❾ Airbnb
- **ポストの主張：** KPIを掲載数→初回予約成功率に変更。ホスト数より「最初の1回が成立したか」を最重視し、写真撮り直し・説明文添削・価格最適化をCSに徹底。
- **判定：概ね正しい ✅**
- **事実：** Brian Chesky含む創業者チームが複数の公開講演やインタビューで語っている戦略。

#### ⑩ サイゼリヤ
- **ポストの主張：** KPIを客単価→回転率＋来店頻度に変更。高い料理を売らないことを徹底して、一回の売上ではなく年間の来店回数を追う。
- **判定：方向性は正しい ✅**
- **事実：** サイゼリヤの低価格・高回転モデルは広く知られた事実。具体的なKPI変更の一次ソースは未確認。

#### ⑪ メルカリ（※ポストでは「サイゼリヤ」と誤記）
- **ポストの主張：** KPIをGMV→初出品完了率に変更。売上より「最初の1品を出品できたか」。UIを極限まで簡単にして出品の面倒くささを排除。
- **判定：概ね正しい ✅（ただしポスト上の表記は「サイゼリヤ」になっている誤記）**
- **事実：** メルカリが出品ハードルを下げることに注力したのは、プロダクトチームが各所で公言している。

## Part 2：パーソナルファイルへのKPI適用

### サービス概要

**サービス名：** 相性転職パーソナルファイル
**URL：** https://per-sonal.co.jp/personalfile/adlp_second/
**運営：** 株式会社パーソナルファイル

**サービス内容：**
- フリーター・高卒・既卒・第二新卒など、スキルがない20代のための未経験転職支援サービス
- 最先端のAIを活用した「キャリアパーソナリティ診断」（200問・80類型）で、パーソナリティをデータ分析し、相性の良い職種や企業を紹介
- スキルや学歴よりも「相性」重視
- 完全無料（企業からの紹介手数料で収益化）

**公表実績：**
- 就職成功者の95%が未経験
- 入社半年後の定着率98%
- 診断結果の納得度94%

**診断の特徴：**
- 設問数：約200問（精度のために多い）
- 分類：80類型（一般的なMBTIの16類型より遥かに細かい）
- 性格・行動傾向、相性の良い働き方、向いている職種、相性の良い上司タイプなど16項目が判明

**収益モデル：** 企業からの紹介手数料（成功報酬型の人材紹介）

### 現在のファネル構造

```
広告接触 → LP訪問 → 個人情報登録 → LINE登録 → 診断URLをLINEで共有 → 診断受検（200問）→ 面談予約 → 面談実施 → 求人紹介 → 応募 → 面接 → 内定 → 入社 → 入社後定着 → 紹介手数料確定（＝売上）
```

## Part 3：KPI議論の経緯と結論

### 議論の流れ

#### ステップ1：最初のKPI提案

11社の「売上の一歩手前をKPIにする」原則をパーソナルファイルに適用。

**最初の提案：** 「診断結果の共有・言語化アクション数」（診断結果をSNSにシェアした、スクショ保存した、自分の言葉で言語化した人の数）

**根拠：** Dropboxの「招待を送った人は定着する」、Duolingoの「10日Streakで離脱しない」と同じ原理。診断結果を自分のものとして受け入れ共有した人は、転職活動をパーソナルファイルと進める確率が飛躍的に上がるはず。

#### ステップ2：ファネル構造の補正

ユーザーからファネルの補正情報が入った。当初は「面談中に診断を受ける」と想定していたが、実際は**診断が面談の前にある**ことが判明。

**修正後のKPI：** 「診断結果の再閲覧率（24時間以内）」

**根拠：** 診断がカウンセラーと会う前に1人で受けるものである以上、診断結果ページをもう一度開いた人は結果が刺さっている証拠。これがSlackの「2,000メッセージ」に相当する「Aha Moment」。

#### ステップ3：ファネル改善の議論①（ミニ診断案）

200問がファネルの中盤にあることが最大のボトルネックと分析。200問の前に個人情報登録→LINE登録という壁が2つあり、好奇心のエネルギーが消耗される。

**提案：** 200問をミニ診断（10〜15問）と本診断に分割し、ミニ診断をLP上で個人情報不要で即受検可能にする。

#### ステップ4：ミニ診断案への反論

ユーザーから反論：「ミニ診断はありきたりすぎて飽きられている。200問の多さを逆に差別化として打ち出すべきでは？"本物の診断はそんなに簡単にはわからない"という訴求の方が刺さる。」

**この反論は正当と判断。** 理由：
1. ターゲットが「本気で転職したい20代」なので、200問に耐えられる人こそ本気の人（天然のクオリフィケーションフィルター）
2. 「努力の正当化（Effort Justification）」により、200問かけた結果の方が心理的価値が高い
3. 「簡単・手軽」で戦うと大手に広告費で負ける。200問・80類型は誰も真似しないポジション

**方針転換：** ミニ診断ではなく、200問を武器として維持する。改善すべきは「200問を受ける意味づけ」の演出。

#### ステップ5：ファネル改善の議論②（壁の移動案）

200問を維持した上で、個人情報登録とLINE登録を診断完了の「後」に移動する案を提案。

**反論：** 「LPに来たばかりの人がいきなり200問やると思う？」→やらない。壁をずらしても、200問に挑む動機がない状態では同じ。

#### ステップ6：最終結論（面談を先にするファネル）

**核心の気づき：** 200問を「受けたい」と思わせる最強の手段は、プロダクトの工夫ではなく、**カウンセラーという人が介在して文脈を作る**こと。

---

### 最終ファネル提案

```
広告接触
  → LP訪問
    → 面談予約（個人情報登録 + LINE登録を兼ねる）★ここが唯一の壁
      → ライト面談（15〜20分）★信頼構築 + 200問を受ける動機づけ
        → 200問受検（面談後〜次回面談前）★信頼関係がある状態で受検
          → 本面談（診断結果の深掘り + 求人紹介）
            → 応募 → 面接 → 内定 → 入社 → 定着 → 売上
```

#### 変更点
1. **診断と面談の順番を逆転：** 面談→診断の順に変更
2. **面談を2回に分割：** ライト面談（信頼構築）→ 本面談（診断結果の深掘り + 求人紹介）
3. **個人情報登録・LINE登録・面談予約を1ステップに統合**

#### なぜこれが機能するか

**①面談予約のハードルは200問より低い：** 「無料で15分相談」は「200問の診断」より遥かにハードルが低い。未経験で転職に不安を抱えている20代にとって、「誰かに話を聞いてもらえる」こと自体に価値がある。

**②カウンセラーが200問を受ける理由を直接伝えられる：** ライト面談でユーザーの状況をヒアリングした上で、「あなたのことをもっと正確に知るために200問の診断を受けてほしい」と個別の文脈で伝えられる。信頼できる人からの推薦＋次回面談という締め切りの存在で、診断完了率が大幅に向上するはず。

**③カウンセラーが「生の声」を先に聞ける：** 診断結果なしで話すからこそ、ユーザーの自己認識と診断結果のズレを本面談で深くカウンセリングできる。現状だと診断結果ありきで面談が始まり、カウンセラーが読み上げ係になるリスクがある。

**④自然なクオリフィケーション：** ライト面談で「この人は本気じゃない」と判断できれば200問を勧めず終了できる。カウンセラーが人力で質のフィルタリングをかけられる。

#### 推定効果

- 現在：LP訪問の約0.6%が面談到達（推計）
- 改善後：面談予約率の大幅向上 + 200問完了率が50〜70%に上昇（現状推定10〜20%）
- 200問を完了できる人だけが本面談に来るので、面談の質は維持

### 最終KPI設計

| KPI | 定義 | 役割 |
|------|------|------|
| ライト面談予約率 | LP訪問→面談予約の転換率 | ファネル入口の効率 |
| **★ライト面談→診断完了率** | **面談後に200問を完了した割合（最重要）** | **サービスへのコミットメントの閾値。Slackの2,000msg、Duolingoの10日Streakに相当** |
| 診断結果の再閲覧率 | 結果を見た後、もう一度見に来たか | Aha Momentの確認 |
| 本面談→応募率 | 2回目面談で求人に応募した割合 | カウンセラーの質の指標 |

**「ライト面談→診断完了率」が、このサービスにおける "売上の一歩手前" のKPI。** ここを追えば、全てが連鎖的に改善する。

### 追加施策案

#### LP上の訴求変更
- 200問の本気の適職診断はサービスの信頼性・差別化ポイントとして見せるが、CTAは面談予約にする
- 「200問のキャリアパーソナリティ診断であなたを80タイプに分析し、本当に相性の良い仕事だけを紹介します。まずは無料面談で、あなたの状況を聞かせてください。」

#### 200問の受検体験改善
- 途中に進捗バー＋褒めを入れる（50問・100問・150問で到達メッセージ）
- 途中保存→LINEリマインド機能（「あなたの診断は63%完了。あと約8分で結果が見られます」）
- 完了時に希少性の演出（「この診断を最後まで受けられたのは登録者の○○%だけです」）

#### LINE上の事前教育メッセージ（面談前に診断を受けさせる際）
1. 「この診断は一般的な適職診断とは全く違います」
2. 「設問数は約200問。でもそれには理由があります。よくある3分の診断は4〜16タイプに分けるだけ。パーソナルファイルは80タイプ。だから200問必要です」
3. 「この診断を受けた方の94%が結果に納得。転職した方の98%が半年後もその職場で活躍しています」
4. 「所要時間は約15〜20分。静かな場所で集中できる時間に受けてください」

#### バイラル設計
- 診断結果のSNSシェアカード自動生成（「あなたは【効率型PD 起業家タイプ】」のビジュアルカード）
- 80類型の細かさが「自分のタイプが珍しい」という発見につながり、シェア動機になる
- 16personalities.comがMBTI16タイプ診断でバイラル成長した手法の応用

## 議論から得られた原則

### 11社のKPI事例から抽出した汎用原則

1. **売上を直接追うと、売上の手前にある「依存・習慣・愛着が生まれる瞬間」を見落とす**
2. **「Aha Moment（この人はもう離脱しない行動閾値）」を特定し、それをKPIにする**
3. **プロダクトの機能だけでは人の行動を変えにくい場合、人が介在して文脈を作ることが最強のレバー**
4. **ファネルの壁は「価値提供の後」に置く。まだ何も体験していない段階でコミットメントを要求しない**
5. **重さ（200問）は弱点にも武器にもなる。コモディティ化した市場では、あえて重くすることが差別化になる**
