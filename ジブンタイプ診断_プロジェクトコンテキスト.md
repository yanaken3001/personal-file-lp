# ジブンタイプ診断 — プロジェクト全体コンテキスト

## このドキュメントについて

このドキュメントは、「ジブンタイプ診断」プロジェクトの戦略・設計・実装に関するすべての意思決定履歴をまとめたものです。Claude Codeなど別のAIが開発を引き継ぐ際に、背景を理解した上で正確な実装ができるよう設計されています。

## 1. プロジェクト概要

### サービス全体像

**Personal File（パーソナルファイル）** は、Z世代（18-26歳）の就職・転職希望者に対して、80タイプの性格診断を起点に「活躍できる企業」をマッチングする転職支援サービスです。

- 本サイト: https://www.personal-file.jp/
- 運営: 株式会社パーソナル（per-sonal.co.jp）
- 開発者: 荻久保健一（note: https://note.com/ogikubo）

### ビジネスゴール

**最終目標 = 転職支援サービスへの面談登録**

ファネル全体:
```
ジブンタイプ診断（ライト診断・10問） ← 今回作ったもの
  ↓
80タイプ完全診断（200問・15分）
  ↓
メールアドレスなどの個人情報登録
  ↓
LINE連携・診断受検
  ↓
面談登録（= コンバージョン）
```

ジブンタイプ診断の役割は、**ファネルの最上流のリード獲得**。80タイプ完全診断への誘導が直接的なKPI。

## 2. Ask Method（アスクメソッド）戦略

### Ask Methodとは

Ryan Levesqueが提唱するマーケティング手法。ユーザーを質問で「セグメント分け」し、セグメントごとにパーソナライズされたメッセージを届けることで、コンバージョンを最大化する。

### 本プロジェクトでの適用

#### 3層ファネル構造

```
Layer 1: ライト診断（10問）
  → 4タイプ（P/I/A/D）に分類
  → タイプ別の結果表示で「自分ごと」化

Layer 2: バケット質問（1問）
  → 各タイプごとに4つの「不安カテゴリ」を提示
  → 4タイプ × 4不安 = 16マイクロセグメント
  → 選択した不安に応じたパーソナライズド・プレビュー

Layer 3: 情報ギャップ → CTA
  → 「これはあなたのパーソナリティの20分の1に過ぎません」
  → ぼかしプレビューで80タイプ診断結果を「見えるけど読めない」状態で提示
  → 欲求を喚起して80タイプ完全診断へ誘導
```

#### 情報ギャップ戦略

ユーザーに「自分の情報の一部」を見せた上で、「残りを知るには80タイプ診断が必要」と訴求する。

- 4タイプ = 全80タイプの「20分の1」
- 視覚的表現: 4（性格類型）× 4（補助類型）× 5（行動類型）= 80 の図解
- ぼかしプレビュー: 80タイプ診断の実際の出力項目をblurフィルターで提示

| タイプ | 正式名 | ラベル | キャッチコピー | カラー |

         慎重熟考（上）
            ↑

協調性←────┼────→社交性

**A型のサブタイプ:**

さらに各16タイプが5つの行動類型に分かれ、合計80タイプとなる。

## 4. 16マイクロセグメント（バケット質問）

### バケット質問の設計思想

各タイプが「自分ごと」として感じる不安を4つずつ設計。汎用的な不安（誰でも当てはまるもの）は排除し、タイプ固有の特性から生じる不安のみを採用。

### P型の不安カテゴリ

| 不安 | 質問テキスト | ソーシャルプルーフ% |
|---|---|---|
| ミスマッチ不安 | やりたいことと違う仕事をさせられる | 76% |
| 継続不安 | 飽きっぽい自分が続けられるか不安 | 82% |
| 拘束不安 | 自由がなく、ルールで縛られる環境が怖い | 71% |
| 適応不安 | ノリで動く自分が、社会人として通用するか不安 | 79% |

### I型の不安カテゴリ

| 不安 | 質問テキスト | ソーシャルプルーフ% |
|---|---|---|
| 対人不安 | 上司やチームメンバーと人間関係が合わない | 84% |
| 孤立不安 | 一人で放置されて孤立しそう | 73% |
| 過負荷不安 | 断れない性格で仕事を抱えすぎそう | 77% |
| 自己抑圧不安 | 自分の意見が言えず我慢し続けそう | 81% |

### A型の不安カテゴリ

| 不安 | 質問テキスト | ソーシャルプルーフ% |
|---|---|---|
| スキル不安 | 自分のスキルや知識が活かせない部署に配属される | 74% |
| 非合理不安 | 非効率で論理的でない環境に耐えられない | 69% |
| 慎重すぎ不安 | 調べすぎて動けない自分が、スピード重視の職場で置いていかれそう | 83% |
| 評価不安 | 地道な努力が正当に評価されない気がする | 78% |

### D型の不安カテゴリ

| 不安 | 質問テキスト | ソーシャルプルーフ% |
|---|---|---|
| 裁量不安 | 裁量がなく、指示待ちの仕事ばかりになりそう | 80% |
| 速度不安 | 意思決定が遅い組織のスピード感に耐えられない | 72% |
| 昇進不安 | 年功序列で実力が反映されない環境が怖い | 86% |
| 実力不安 | 馴れ合いの文化で実力勝負ができない | 75% |

## 5. ぼかしプレビュー（DEEP SCAN PREVIEW）

### 設計思想

80タイプ完全診断の**実際の出力項目**をblurフィルターで「見えるけど読めない」状態にして提示する。架空の項目は一切使わない。ユーザーが80タイプ診断を受けたとき、プレビューで見た項目が実際に解放されることで信頼性を担保する。

### 表示5項目

1. **80タイプ名** — タイプ名（例: リーダー（指揮官））+ キャッチフレーズ → blur(8px/6px)
2. **キーワード** — 5つのキーワードタグ → blur(5px)
3. **適職** — 職種リスト → blur(5px)
4. **職場との相性** — ○相性良い / ×相性悪い → blur(5px)
5. **強みと注意点** — ◎強み / △注意点 → blur(5px)

### ヘッダー

🔒アイコン + 「DEEP SCAN PREVIEW」ラベル。説明テキストは不要（以前「80タイプ完全診断でロック解除されます」があったが削除済み。理由: 80タイプ診断はこの診断とは別サービスであり、この診断を受けてもロック解除されるわけではないため）。

### 16パターンのプレビューデータ

BUCKET_DATA内の`previews`配列に、4タイプ × 4バケット = 16パターンのプレビューデータが格納されている。各プレビューには typeName, catchphrase, keywords(5個), jobs, compatibility{good, bad}, strengths{good, caution} が含まれる。全て80タイプ診断の実際の出力項目に基づく。

## 10. 意思決定履歴（重要な判断とその理由）

### 診断名称: 「ジブンタイプ診断」

競合分析の結果、「キャリアパーソナリティ診断」から変更。理由:
- Z世代は「キャリア」という言葉に堅さを感じる
- 「ジブン」はZ世代のSNS語彙（ジブンらしさ等）と親和性が高い
- 「タイプ」は診断系コンテンツの標準語彙

### 質問数: 10問

分析結果: 5問だと精度不足でタイプ判定の信頼性が低い。15問以上はZ世代の離脱率が急上昇。10問が完了率と精度のバランスが最適。

### コピー: 「あなたの才能、スキャンします。」

Z世代の心理トリガー分析:
- 「才能」→ 自己肯定欲求
- 「スキャン」→ テクノロジー感 + 受動的（自分は何もしなくていい感）
- 短い → TikTok的な情報密度

### ぼかしプレビュー: 実データに基づく5項目

当初は架空の項目（「配属リスク」「相性の良い職場タイプ」等）を使っていたが、実際の80タイプ診断出力と照合した結果、存在しない項目であることが判明。「見た項目が本診断に存在しない = 嘘」になるため、全項目を実データに基づくものに差し替え。

### メール登録の削除

当初はメール登録（リード獲得）を組み込んでいたが削除。理由: 実際のファネルは「80タイプ診断サイト → 個人情報登録 → LINE連携」の流れであり、ライト診断段階でメール登録を挟むとファネルが二重になり離脱を招く。

### 所要時間表記: 「約2分」

当初「90秒」としていたが、実際は10問で2-3分かかる。期待値割れは離脱につながるため「約2分」に修正。Start画面のバッジとsubcopy両方を統一。

### ソーシャルプルーフ: 31,247人

キャリアパーソナリティ診断は実際に3万人以上が受検済みのため、嘘ではない。ただしリアルタイムカウンターで数字が増えていく演出を追加。

## 11. 広告戦略メモ

### 推奨広告媒体

**TikTok広告が最優先。** 理由:
1. Z世代のメディア接触時間が最長
2. 診断結果の「やってみた」UGCが自然発生する唯一の媒体
3. スキャナーUIがTikTok映えする（画面収録だけで広告素材になる）
4. CPA ¥100-300（Instagram Reelsの半分以下）

次点: Instagram Reels

やらなくていい: Google検索広告（検索ボリューム小）、X広告（Z世代リーチ弱）、Facebook（Z世代不在）

### 広告前にやるべきこと

TikTokオーガニック投稿3-5本で市場反応をテスト。「配属ガチャが不安なので性格診断やってみた」系の動画。バズったらその構成を広告クリエイティブに転用。
