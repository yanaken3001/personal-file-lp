# /develop — フロントエンド＆バックエンド開発部を起動

実装・コーディング・インフラに関する指示を処理します。

## 起動エージェント
- **yuto**（FEリード）— アーキテクチャ・コンポーネント実装・技術選定
- **mei**（アニメーション）— 演出実装・インタラクション・アニメーション
- **kai**（バックエンド）— API・DB・認証・ユーザー管理
- **nao**（インフラ）— Vercel・パフォーマンス・OGP生成

## ルーティング判断
- 「実装」「コーディング」「コンポーネント」「React」「Next.js」→ yuto起動
- 「アニメーション」「トランジション」「演出実装」→ mei起動
- 「API」「データベース」「認証」「ユーザー管理」→ kai起動
- 「デプロイ」「パフォーマンス」「OGP」「Vercel」→ nao起動
- 複合タスク → 関連する複数名を並列起動

## 起動手順
1. 該当するエージェント定義ファイルを読み込む
2. `guidelines/tech-standards.md` を必ず読み込む
3. 既存コード参照が必要な場合 `index.html` を読み込む
4. Agent toolでサブエージェントとして起動する

## 使用例
```
/develop 診断ロジックをTypeScriptに移植して
/develop カード獲得のアニメーションを実装して
/develop ユーザー管理のDB設計をして
/develop OGP動的生成の仕組みを作って
```
