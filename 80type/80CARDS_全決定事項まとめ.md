# ■ 5. カードコレクション ＆ 相性システム（確定）

## 設計の核心：「コンテンツ全開放＋機能ロック」

診断結果のコンテンツ（タイプ名・特徴・強み・注意点）は全部無料で見せる。ロックするのは「友達との相性を調べる」「カードを集める」という**機能**のみ。

| | 無料（LINE不要） | LINE連携後に解放 |
|---|---|---|
| 対象 | **コンテンツ**（読むもの） | **機能**（使うもの） |
| 具体的に | タイプ名、キャラ画像、キャッチコピー、特徴説明、強み、注意点、相性TOP3チラ見せ | マイフォルダ、個別シェア、相性診断、LINE通知、相性カード生成 |
| 理由 | バイラルの燃料。スクショ＆シェアされて認知が広がる | LINE登録の動機。「友達との相性」という欲求で登録させる |

**なぜコンテンツを隠さないか：**
- 結果を隠すとシェアする素材がなくなり、バイラルが死ぬ
- 16PersonalitiesもLove Character 64も結果は全開放でバズしている
- 注意点も見せることで「ちゃんと分析してくれてる」という信頼性が上がる
- 信頼性が上がるからこそ「相性」にも信頼が生まれ、LINE登録の動機になる

## 2つのバイラルループ

```
ループ①：SNS拡散ループ（LINE不要）
  Aが結果をSNSにシェア → Bが「面白そう」で診断 → Bも結果をシェア → C,D,E...
  特徴：広く浅い。認知を広げる。インフルエンサーが起爆剤

ループ②：相性ループ（LINE必須）
  AがLINE連携 → 友達BにLINEで個別リンクを送る → Bが診断→LINE連携 → 相性判明
  → BもCにリンクを送る → ...
  特徴：狭く深い。1対1で連鎖。エンゲージメントが高い
```

この2つは相互に加速する。①で認知が広がり→②に入りやすくなる。②で生まれた相性カードが→①でSNSシェアされる。

## ユーザー認証方式：LINE連携必須

**LINE連携を必須にする理由：**
- ユーザーIDが確実に取れる（端末変更・Cookie削除の影響なし）
- カード追加時にLINEプッシュ通知が送れる
- マネタイズ導線（ナーチャリング）が診断完了と同時に確保できる
- フォルダ＝LINEアカウントに紐づくため「永久保存」が自然に成立
- 個別シェアリンクの生成に「誰のリンクか」の紐づけが必要

## スコア算出ロジック
- 公式データの出現回数：共感回数×1.0 ＋ 補完回数×1.2

- 表示時は2者の平均スコア：displayScore = Math.round((MATRIX[typeA][typeB] + MATRIX[typeB][typeA]) / 2)
